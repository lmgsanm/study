## 4.1	数组

数组是具有固定长度且拥有零个或者多个相同数据类型元素的序列。

数据组的长度固定。

数组的长度必须是常量表达式，也就是说，这个表达式的值在程序编译时就可以确定。

#### 示例1

完整代码

```
package main

import "fmt"

type Currency int

func main() {
	const (
		USD Currency = iota
		EUR
		GBP
		RMB
	)
	symbol := [...]string{USD: "$", EUR: "E", GBP: "G", RMB: "￥"}
	fmt.Println(RMB, symbol[RMB])
}
```

输出结果

```
3 ￥
```

- 如果一个数组的元素类型是可比较的，那么这个数组也是可以比较的：
  - 使用==来比较两个数据，比较的结果是两边元素的值是否完全相同	
  - 使用!=来比较两个数据是否不同

#### 示例2

完整代码

```
package main

import "fmt"

func main() {
	a := [2]string{"tom", "jerry"}
	b := [...]string{"tom", "jerry"}
	c := [2]string{"tom", "help"}
	d := [3]string{"tom", "help", "sanm"}
	fmt.Println(a == b, b == c, a == c, a != c)
	fmt.Println(d)
	// fmt.Println(c == d)	#invalid operation: c == d (mismatched types [2]string and [3]string)

}
```

输出结果：

```
true false false true
[tom help sanm]
```

当调用一个函数的时候，每个传入的参数都会创建一个副本，然后赋值给对应的函数变量，所以函数接受的是一个副本，而不是原始的参数。使用这种方式传递大的数组会变更很低效，并且在函数内部对数组的任何修改都仅影响副本，而不是原始数组。这种情况下，Go把数组和其它的类型都看成值传递。

可以显示地传递一个数组的指针给函数，这样在内部对数组的任何何必都会反映到原始数组上面。

#### 示例

将一个数组为[32]byte的元素清零的2种方式

```
func zero(ptr *[32]byte) {
	for i := range ptr {
		ptr[i] = 0
	}
}
```

```
func zero(ptr *[32]byte) {
	*ptr = [32]byte{}
}
```

## 4.2	slice

slice表示一个拥有**相同类型元素**的**可变长度**的序列。

slice是一种轻量级的数据结构。可以用来访问数组的部分或者全部元素，而这个数据称为slice的底层数组

- slice的三个属性
  - 指针：指针指向数组的第一个可以从slice中访问的元素，这个元素并不一定是数组的第一个元素
  - 长度：指slice中的元素个数，它不能够超过slice的容量（函数len）
  - 容量：容量的大小通常是从slice的起始元素到底层数组的最后一个元素间元素的个数（函数cap）

一个底层数组可以对应多个slice，这些slice可以引用数组的任何位置，彼此之间的元素还可以重叠

slice操作符s[i:j]（其中0 =< i =< j =< cap(s)）创建了一个新的slice，这个slice引用了序列s中从i到j-1索引位置的所有元素
