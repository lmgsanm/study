## 2.3变量

​	var name type = expression

​	类型和表达式部分可以省略一个，但是不能都省略。如果类型活力，它的类型将由初始化表达式决定； 如果表达式省略，其初始值对应于类型的零值：

- 对于数字是0

- 对于布尔值是false

- 对于字符串是“”

- 对于接口和引用类型（slice、指针、map、通道、函数）是nil

- 对于一个像数组或结构体的复合类型，零值是其所有元素的零值

  

  Go里面不存在未初始化变量

  

  Go程序员经常花费精力来使复杂类型的零值有意义，以便变量一开始就处于一个可用状态

  

  可以声明一个变量列表，并选择使用对应的表达式列表对其初始化。忽略类型允许声明多个不同类型的变量。

```
package main

import "fmt"

func main() {
	var i, j, k int
	var b, f, s = true, 2.3, "four"
	fmt.Println(i)
	fmt.Println(j)
	fmt.Println(k)
	fmt.Println(b)
	fmt.Println(f)
	fmt.Println(s)
}
```

初始值设定可以是字面量值或者任意的表达式：

- 包级别的初始化在main开始之前进行
- 局部变量初始化和声明一样在函数执行期间进行



变量可以通过调用返回多个值的函数进行初始化

### 2.3.1 短变量声明

name := expression

name的类型由expression的类型决定

在局部变更的声明和初始化中主要使用短声明

var声明通常是为那些跟初始化表达类型不一致的局部变量保留的，或者用于后面才对变更赋值以及变量初始化值不重要的情况

只有当它们对于可读性有帮助的时候才使用多个初始化表达式来进行变更声明，例如短小且天然一组的for循环的初始化

:=表示声明，而=表示赋值。一个多变量的声明不能和多重赋值搞混

与普通的var声明有类似，短变量声明也可以用来调用返回两个多个值的函数



短变量声明**不需要声明所有在左边的变量**。如果一些变量在同一个词法块中声明，那么对于那些变量，短声明的行为等同于赋值

只有在同一个词法块中已经存在变量的情况下，短声明的行为才和赋值操作一样，外层的声明将被忽略。

### 2.3.2 指针

变量是存储值的地方

指针是一个变量的地址。一个指针指示值所保存的位置。不是所有的值都有地址，但是所有的变量都有。使用指针，可以在无须知道变量名字的情况下，间接读取或更新变量的值。



如果一个变更声明为var x int，表达式&x（x的地址）获取一个指向整型变更的指针，它的类型是整型指针（*int）。

如果值叫作p，我们说p指向x，或者p包含x的地址。p指向的变量写成*p。表达式

```
package main

import "fmt"

func main() {
	x := 1
	p := &x
	fmt.Println(x)	//1
	fmt.Println(*p)	//1

	*p = 2
	fmt.Println(x)	//2
}
```

每一个聚合类型变量的组成（结构体的成员或数组中的元素）都是变量，所以也有一个地址。

变量有时候使用一个**地址化**的值。代表变量的表达式，是唯一可以应用取地址操作&的表达式。

指针类型的零值是nil。测试p != nil，结果是true说明p指向一个变量。

指针是可以比较的，两个指针当且仅当反指向同一个变量或者两者都是nill的情况下才相等。

```
package main

import "fmt"

func main() {
	var x, y int
	fmt.Println(&x == &x, &x == &y, &x == nil)	//true false false
}
```

函数返回局部变更的地址是非常安全的。因为一个指针包含变量的地址，所以传递一个指针参数给函数，能够让函数更新间接传递的变量值 。

```
package main

import "fmt"

func incrs(p *int) int {
	*p++
	return *p
}
func main() {
	v := 1
	incrs(&v) //2
	fmt.Println(incrs(&v))	//3
}
```

指针对于flag包是很关键的，它使用程序命令行参数来设置整个程序内某些变量的值。

```
package main

import (
	"flag"
	"fmt"
	"strings"
)

var n = flag.Bool("n", false, "omit trailing newline")
var sep = flag.String("s", " ", "separator")

func main() {
	flag.Parse()
	fmt.Print(strings.Join(flag.Args(), *sep))
	if !*n {
		fmt.Println()
	}
}
```

### 2.3.3 new函数

另外一种创建变量的方式是使用内置的new函数。

表达式new(T)创建一个未命名的T类型变量，初始化为T类型的零值，并返回其地址（地址类型为*T)

```
package main

import "fmt"

func main() {
	s := new(int)
	fmt.Println(*s)	//0

	*s = 3
	fmt.Println(s)	//0xc00000a0f8
	fmt.Println(&s)	//0xc000068060
	fmt.Println(*s)	//3
}
```

每一次调用new返回一个具有唯一地址的不同变量

```
package main

import "fmt"

func main() {
	s := new(int)
	t := new(int)
	fmt.Println(s == t)		//false
}
```

有一个例外：两个变量的类型不携带任何信息且是零值，例如struct{}或[0]int，当前的实现里面，它们有相同的地址

new是一个预声明的函数，不是一个关键字，所以它可以重定义为另外的其它类型。例如：

func delta(old,new int) int { return new - old}

在delta函数内，内置的new函数是不可用的。

### 2.3.4 变量的生命周期

生命周期指在程序执行过程中变更存在的时间段

- 包级别变量的生命周期是整个程序的执行时间
- 局部变量有一个动态的生命周期：每插声明语句时创建一个新的实体，变量一直生存到它变得不可访问，这里它占用的存储空间被回收

函数的参数和返回值也是局部变量，它们在其闭包函数被调用的时候创建。

每一个包级别的变量，以及每一个当前执行函数的局部变量，可以作为追溯该变量的路径的源头，通过指针和其它方式的引用可以找到到变量。如果变量的路径不存在，那么变更变得不可访问。

# 2.4 赋值

赋值语句用来更新变更所指的值

### 2.4.1 多重赋值

​	另一种形式的赋值是多重赋值，它允许几个变量一次性被赋值。在实际更新变量前，右边所有的表达式被推演。

示例1：计算两整数的最大公约数

```
package main

import "fmt"

func gcd(x, y int) int {
	for y != 0 {
		x, y = y, x%y
	}
	return x
}
func main() {
	x, y := 4, 8
	fmt.Println(gcd(x, y))	//4
}
```

示例2：计算斐波那契数列的第n个数

```
package main

import "fmt"

func fib(n int) int {
	x, y := 0, 1
	for i := 0; i < n; i++ {
		x, y = y, x+y
	}
	return x
}
func main() {
	n := 10
	fmt.Println(fib(n))	//55
}
```

​	从风格上考虑，如果表达式比较复杂，则避免使用多重赋值的形式； 一系列独立的语句更易读

​	这类表达式（例如一个有多个 返回值的函数调用）产生多个值。当在一个赋值语句中使用这样的调用时，左边的变量个数需要和函数的返回值一样多。

​	像变量声明一样，可以将不需要的值赋给空标识符。

### 2.4.2 可赋值性

​	赋值语句是显式形式的赋值，但是程序中有很多地方的赋值是隐式的：

- 一个函数调用隐式的将参数的值赋给对应参数的变量
- 一个return语句隐式地将return操作数赋值给结果变量
- 复合类型的字面表达式，如slice
- map和通道的元素尽管不是普通变量，但是他们遵循相似的隐式赋值

```
package main

import "fmt"

func main() {
	medals := []string{"gold", "silver", "bronze"}
	fmt.Println(medals)	//[gold silver bronze]

	medals[0] = "test0"
	fmt.Println(medals)  //[test0 silver bronze]

	medals[1] = "test1"
	fmt.Println(medals) //[test0 test1 bronze]

	medals[2] = "test2"
	fmt.Println(medals)  //[test0 test1 test2]
}
```

​	不管是显式赋值还是隐式赋值，如果左边的（变量）和右边的（值）类型相同，它就是合法的。通俗的说，赋值只有在值对于**变量类型是可赋值**的时候才合法。

​	类型必须是精确匹配，nil可以被赋给任何接口的变量或引用。

​	两个值使用==或!=进行比较与可赋值性相关：在任何比较中，第一个操作数相关于第二个操作数的类型必须是可赋值的，或者可以反过来赋值。
